===========================================================================
PLANO DE IMPLEMENTACAO - PARTIAL OPTIMIZER (ETAPAS A A D)
PPMC / ILS + MATHEURISTICA INSPIRADA NO IRMA
===========================================================================

Objetivo geral
  Evoluir a heuristica atual

    GRASP -> VND -> ILS

  para uma arquitetura hibrida com intensificacao exata por subproblemas:

    GRASP -> VND -> ILS -> PARTIAL OPTIMIZER (CPLEX)

  A ideia central eh:
    - usar o ILS para diversificacao e refinamento local
    - usar o CPLEX apenas em subregioes da solucao
    - manter o modelo pequeno e controlado
    - integrar a melhoria parcial de volta na solucao global


===========================================================================
VISAO GERAL DA ARQUITETURA
===========================================================================

Camada 1 - ILS
  - construcao inicial
  - busca local VND
  - perturbacao estrutural
  - retorno da melhor solucao encontrada

Camada 2 - Partial Optimizer
  - seleciona uma subregiao da solucao
  - monta um submodelo reduzido
  - resolve via CPLEX
  - integra a solucao parcial de volta

Camada 3 - Reducoes
  - R1 para filtrar atribuicoes improvaveis
  - R2 para restringir candidatos a mediana
  - entram depois que a infraestrutura do subproblema estiver estavel


===========================================================================
ETAPA A - ESQUELETO FUNCIONAL MINIMO
===========================================================================

Objetivo
  Provar que conseguimos:
    1. selecionar uma subregiao da solucao atual
    2. montar um MIP pequeno com CPLEX
    3. resolver esse subproblema
    4. integrar a nova solucao corretamente na estrutura Solution

Escopo da Etapa A
  - sem R1
  - sem R2
  - sem warm start
  - sem capacidade residual sofisticada
  - sem loop de clusters otimizados / nao-otimizados
  - sem reinicio do ILS

Regras da primeira versao
  - chamar o partial optimizer apenas uma vez apos o ILS
  - escolher um cluster de referencia
  - liberar uma subregiao por clusters vizinhos
  - considerar todos os nos dessa subregiao como clientes livres
  - considerar todos os nos da subregiao como candidatos a mediana
  - resolver e aceitar apenas se melhorar

Selecao do cluster de referencia
  Usar score hibrido do cluster:

    score(m) = 0.7 * custo_total_cluster(m)
             + 0.3 * custo_medio_cluster(m)

  Escolher a mediana aberta com maior score.

Selecao da vizinhanca
  Dada a mediana de referencia:
    - ordenar as demais medianas abertas por distancia a ela
    - ir adicionando clusters vizinhos ate acumular aproximadamente omega nos
    - respeitar um minimo min_clusters

  Parametros iniciais:
    - omega = 300
    - min_clusters = 5

Subproblema da Etapa A
  Conjuntos:
    - free_medians = medianas dos clusters livres
    - free_nodes = clientes atualmente alocados a essas medianas
    - p_sub = numero de clusters livres

  Variaveis:
    - x[i][j] binaria: cliente livre i alocado ao candidato j
    - y[j] binaria: no j aberto como mediana na subregiao

  Funcao objetivo:
    min soma d(i,j) * x[i][j]

  Restricoes:
    (1) cada cliente livre eh alocado exatamente uma vez
    (2) exatamente p_sub medianas sao abertas na subregiao
    (3) x[i][j] <= y[j]
    (4) capacidade:
        soma q[i] * x[i][j] <= Q[j] * y[j]

Integracao da solucao parcial
  - copiar a solucao atual
  - substituir as medianas livres antigas pelas retornadas pelo solver
  - atualizar as atribuicoes apenas dos free_nodes
  - manter o restante fixo
  - reavaliar a solucao com Evaluator
  - aceitar somente se viavel e melhor

Validacao obrigatoria
  A Etapa A deve incluir:
    - validacao pos-integracao com Evaluator
    - comparacao do custo extraido com o custo recalculado
    - asserts ou logs para detectar inconsistencias

Log estruturado recomendado
  Cada chamada ao CPLEX deve imprimir algo como:

    [PARTIAL_OPT] ref_cluster=7 | free_clusters=5 | free_nodes=142
                  | candidate_medians=142 | vars_x=1847
                  | fo_before=95034 | fo_after=94890
                  | delta=-144 | time=2.3s | status=OPTIMAL

Arquivos a criar
  - src/partial_optimizer.h
  - src/partial_optimizer.cpp

Interface inicial sugerida

  class PartialOptimizer {
  public:
      PartialOptimizer(const Instance& instance,
                       const DistanceMatrix& dm,
                       const Evaluator& evaluator,
                       int omega = 300,
                       int min_clusters = 5,
                       double time_limit_s = 30.0);

      bool optimize(Solution& solution);
  };


===========================================================================
ETAPA B - REDUCAO R1 (VERSAO SIMPLIFICADA PRIMEIRO)
===========================================================================

Objetivo
  Reduzir o numero de variaveis x[i][j] do subproblema para permitir
  subregioes maiores ou tempos menores de CPLEX.

Versao simplificada de R1
  Para cada cliente livre i:
    - ordenar candidatos j por distancia crescente
    - manter apenas os t candidatos mais proximos

  Sugestao inicial:
    - t = min(15, numero de candidatos)

Vantagens da versao simplificada
  - facil de implementar
  - robusta
  - grande reducao do modelo
  - suficiente para medir ganho inicial

R1 mais fiel ao Stefanello
  Depois da versao simplificada, testar uma variante inspirada no artigo:
    - filtrar candidatos por distancia e capacidade
    - usar fator alpha para controlar agressividade da reducao

Parametros iniciais para B
  - alpha_R1 = 2.0
  - t = 15

Resultados esperados
  - menor numero de variaveis x
  - menor tempo do solver
  - possibilidade de usar subregioes maiores


===========================================================================
ETAPA C - INTEGRACAO COMPLETA COM O ILS
===========================================================================

Objetivo
  Transformar o partial optimizer em uma fase sistematica do algoritmo.

Primeira integracao
  Fluxo inicial:

    s = construcao
    s = VND(s)
    s_best = ILS(s)
    s_best = PARTIAL_OPTIMIZER(s_best)
    retornar s_best

Segunda integracao
  Se o partial optimizer melhorar a solucao:
    - atualizar s_best
    - opcionalmente rodar um ILS curto a partir da solucao melhorada

  Exemplo:
    - NumIterMax reduzido
    - ou tempo limitado

Versao completa com clusters otimizados
  Quando a infraestrutura estiver estavel:
    - marcar clusters como otimizados / nao-otimizados
    - ao melhorar uma subregiao, remarcar seus vizinhos como nao-otimizados
    - repetir ate esgotar subregioes promissoras

  Fator de expansao sugerido:
    - h = 1.8

Resultado esperado
  - intensificacao sistematica apos o ILS
  - reaproveitamento das melhores solucoes encontradas
  - melhor aproximacao ao desenho do IRMA


===========================================================================
ETAPA D - REFINAMENTOS E ESCALABILIDADE
===========================================================================

Objetivo
  Melhorar robustez, desempenho e proximidade com o estado da arte.

Itens previstos

1. Warm start no CPLEX
  - passar a solucao atual como incumbente
  - acelerar a convergencia do subproblema

2. Ajuste dinamico do tamanho da subregiao
  - adaptar omega conforme tempo real observado
  - se solver estiver rapido, liberar mais clusters
  - se solver estiver lento, reduzir subregiao

3. R2 para candidatos a mediana
  - restringir candidatos a mediana a nos proximos da mediana atual
  - reduzir ainda mais o submodelo

4. Capacidade residual mais sofisticada
  - quando o desenho da subregiao permitir clientes externos associados
    a medianas livres, descontar carga fixa corretamente

5. Multi-call partial optimization
  - chamar varios subproblemas em sequencia
  - controlar por orcamento de tempo total

6. Polishing / local exact search
  - usar CPLEX como polisher final em uma subregiao pequena
  - opcionalmente acoplar outras formulacoes

7. Instrumentacao experimental
  - logs detalhados por subproblema
  - estatisticas de ganho medio por chamada
  - taxa de melhoria
  - tempo por subproblema

ORDEM DE IMPLEMENTACAO RECOMENDADA
===========================================================================

1. Etapa A
  Fazer primeiro o esqueleto funcional, sem reducoes.

2. Etapa B
  Adicionar R1 simplificado e medir impacto no tamanho do modelo.

3. Etapa C
  Integrar o modulo ao fluxo do ILS e estabilizar a chamada apos convergencia.

4. Etapa D
  Fazer refinamentos so depois que as etapas anteriores estiverem corretas.


===========================================================================
RISCO TECNICO PRINCIPAL
===========================================================================

O maior risco nao eh o CPLEX, e sim a integracao correta da solucao parcial.

Ao atualizar a solucao global, eh preciso garantir:
  - medianas consistentes
  - atribuicoes consistentes
  - cargas corretas
  - custo correto
  - viabilidade final

Portanto, toda iteracao do partial optimizer deve terminar com:
  - validate()
  - recomputacao de custo/carga quando necessario
  - logs para depuracao
